# Frontend – Known Issues

Issues found in the ACBU frontend web application (Next.js 16/React 19, TypeScript, shadcn/ui, Tailwind). Add new items below as numbered list entries.

---

## Critical

1. **Auth state rehydration without cookie validation** – contexts/auth-context.tsx: On page reload, `isAuthenticated` may be restored from `sessionStorage` before the `getMe()` API call completes. UI flashes authenticated content (balance, wallet actions) for a user whose session cookie is actually expired.
2. **Passcode stored in JS-accessible memory for full session lifetime** – lib/passcode-manager.ts: The passcode for Stellar wallet decryption lives in a module-level variable. Any XSS payload that executes in the same origin can read it via `window.__passcode` or by monkey-patching the module's exports at runtime.
3. **`temp_passphrase` in sessionStorage during wallet setup** – app/wallet/: The wallet creation flow stores a temporary passphrase in sessionStorage for the user to confirm. If the user's device is compromised during setup, the passphrase is exfiltratable via XSS before the wallet is even created.
4. **Missing Content-Type or X-Content-Type-Options in API client** – lib/api/client.ts: The fetch wrapper may not set `Content-Type: application/json` on POST bodies. An endpoint that sniffs MIME type could misinterpret the request, or a misconfigured CDN could serve JSON as HTML, triggering XSS.
5. **No SRI (Subresource Integrity) on external scripts** – app/layout.tsx: If any third-party script tags (analytics, chat widget, status page) are loaded, they lack `integrity` hashes. A compromised CDN can inject malicious JavaScript with no browser-level detection.
6. **Open redirect on sign-in callback** – app/auth/signin/page.tsx: After successful sign-in, the redirect path may be read from `router.query.redirect` without validating that the target is a same-origin path. An attacker can craft a link like `/auth/signin?redirect=https://evil.com` to phish users after login.
68. **OAuth callback state parameter not validated against session** – app/auth/oauth/callback/page.tsx: The OAuth2 state parameter may not be compared to the value stored in the user's session before the redirect. CSRF attacks on the OAuth callback can link an attacker's social account to the victim's ACBU account.
69. **`localStorage` used for sensitive wallet metadata** – app/wallet/: Wallet metadata (addresses, balances, last transaction hashes) may be persisted to localStorage without encryption. Any browser extension with storage permission can read the user's complete transaction history and wallet addresses.
70. **Video/audio autoplay on data-saver connections** – components/onboarding/: Onboarding videos or animations may autoplay without checking `navigator.connection.saveData` or `prefers-reduced-motion`. Users on metered connections incur unexpected data charges.
71. **`dangerouslySetInnerHTML` used on user-generated content without sanitization** – components/chat/, app/support/: If chat messages, support ticket descriptions, or any user-generated text is rendered with `dangerouslySetInnerHTML`, stored XSS payloads execute in the context of other users.
72. **JWT access token stored in `localStorage` and sent on every API call** – lib/api/client.ts: The access token in localStorage is accessible to any script on the page. A successful XSS attack exfiltrates the token, granting full account access until the token expires (potentially hours).

## High

7. **Session expiry not communicated to user before critical actions** – hooks/: The user can navigate to `/send`, fill a transfer form, and click "Confirm" only to get a 401 because the session expired 30 seconds ago. There is no proactive session-expiry warning or token refresh before write operations.
8. **2FA challenge token persisted in sessionStorage on 2FA page refresh** – app/auth/2fa/page.tsx: (Partially addressed) — The 2FA page clears the challenge token on page refresh when passcode is missing, but if the passcode IS present (user went back then forward), the token survives a refresh. The flow should always require a fresh challenge.
9. **Stellar transaction submission without fee polling** – app/send/page.tsx, app/mint/page.tsx: The frontend may submit a Stellar transaction with a hardcoded fee. During network congestion, the fee is too low and the TX hangs in `pending` forever with no feedback to the user.
10. **Burn tab on `/currency` submits without `blockchain_tx_hash`** – app/currency/page.tsx: The burn flow on the currency page calls `POST /burn/acbu` directly without the Stellar `blockchain_tx_hash` that `app/burn/page.tsx` builds. The backend may reject the burn silently or accept it without on-chain proof.
11. **Wallet balance stale after external (non-UI) transactions** – hooks/use-balance.ts: `useBalance()` fetches once on mount and on manual `refresh()`. If the user receives funds from another sender, the UI balance never updates until the user navigates away and back or triggers a refresh themselves.
12. **No confirmation step for IRREVERSIBLE blockchain operations** – app/burn/page.tsx, app/send/page.tsx: Some critical operations (burning ACBU, sending to external addresses) may skip a confirmation modal. A mis-click sends funds irreversibly with no second chance.
13. **Mint `currency_preference` dropdown is cosmetic** – app/currency/page.tsx: The `mintSource` dropdown (USDC Ethereum / USDC Polygon) is rendered but the API only uses `currency_preference: 'auto'`. The user's selection is ignored, and the mint executes on whatever network the backend defaults to.
14. **International transfer preview rate uses stale or wrong data** – app/currency/page.tsx: The international tab multiplies ACBU by the local currency rate from `/rates`, but the actual cross-border rate includes intermediary and settlement fees that the frontend doesn't know about. The preview is misleadingly optimistic.
15. **Error boundary `error.tsx` does not log errors to backend** – app/error.tsx, app/global-error.tsx: React error boundaries catch crashes but only render a fallback UI. The error detail (component stack, route, user ID) is never POSTed to an error-tracking endpoint, leaving production crashes invisible.
73. **No `loading.js` or Suspense boundary on the `/send` page** – app/send/page.tsx: The send money page fetches balance and rates on mount but has no Suspense boundary. Users on slow connections stare at a blank screen with no loading indicator.
74. **Form input loses focus on every keystroke due to state-update re-render** – app/mint/page.tsx, app/send/page.tsx: Input components may be defined inline in the parent component's render function, causing React to unmount and remount the input on every keystroke. The keyboard dismisses on mobile and the cursor jumps to the end.
75. **`useBalance` refetch interval not cleared on page unmount** – hooks/use-balance.ts: The balance polling interval is set with `setInterval` but the cleanup function in `useEffect` may have a stale closure, causing the interval to continue running after the user navigates away.
76. **Dark mode flash on page load (FOIT – flash of incorrect theme)** – app/layout.tsx: The theme is applied via a `useEffect` after hydration. Before the effect runs, the page renders in light mode (default) even if the user's preference is dark, causing a jarring white flash.
77. **Next.js `Image` with `priority` missing on above-the-fold hero/logo** – app/layout.tsx, app/page.tsx: The main logo and hero image lack `priority` attribute. The browser defers loading the LCP (Largest Contentful Paint) image until after JavaScript executes, adding 500ms+ to LCP.
78. **`fetch` polyfill or `ky` client does not set `Accept-Encoding: gzip`** – lib/api/client.ts: The API client may not request compressed responses. The backend serves uncompressed JSON to clients that could have accepted gzip, doubling bandwidth usage on mobile.
79. **No `preconnect` or `dns-prefetch` for API origin** – app/layout.tsx: The page does not preconnect to the API domain. The TCP+TLS handshake for the first API call starts only after JavaScript executes, adding 100-300ms to the first data fetch.
80. **`shadcn/ui` components imported from barrel index causing tree-shaking failure** – components/ui/: Importing from `@/components/ui` barrel file pulls all UI components into every page that imports any one component, bloating the per-route JavaScript bundle.
81. **No `useCallback` on event handlers passed to memoized child components** – app/send/, app/mint/: Event handlers defined as arrow functions in the parent render create new function references on every render. `React.memo` on child components is defeated, and every child re-renders on every keystroke.
82. **`tabIndex` and `aria-*` props on interactive elements break when `disabled` is toggled** – components/ui/button.tsx: When a button transitions from `disabled` to enabled, the aria attributes and tabindex may not update atomically. Screen readers announce the button as disabled even after it becomes clickable.
83. **`<select>` element styled with Tailwind loses native mobile picker behavior** – components/ui/select.tsx: Custom-styled select dropdowns override the native OS picker on iOS/Android. Users lose the touch-optimized scroll wheel and accessibility features of the native control.
84. **No `<Suspense>` boundary with fallback for the i18n translations loading** – app/[locale]/layout.tsx: If translations are loaded asynchronously (e.g., from a CDN or dynamic import), the page renders with missing translation keys briefly. Users see raw key strings like `"send.title"` before translations arrive.
85. **Page transition animation uses `requestAnimationFrame` without fallback for low-power devices** – app/layout.tsx, components/page-transition.tsx: Page transition animations run at 60fps on powerful devices but may cause jank or 2-second freezes on budget Android devices. Should detect device capability and skip animations.
86. **No `crossorigin="anonymous"` on `<video>` or `<audio>` elements loading external media** – components/onboarding/: Video/audio from external CDNs without `crossorigin` attribute cannot have their frames captured to canvas. Any screenshot or thumbnail generation feature will fail with a tainted canvas error.
87. **`navigator.clipboard.writeText()` called without fallback for insecure contexts** – hooks/use-clipboard.ts: `navigator.clipboard` API is available only in secure contexts (HTTPS). When the app is loaded over HTTP (local development, staging without TLS), clipboard copy silently fails with no user feedback.

## Medium

16. **`mintSource` dropdown options don't match backend-supported networks** – app/currency/page.tsx: The dropdown shows "USDC Ethereum" and "USDC Polygon" but the backend mint endpoint may only support Stellar USDC. Selecting a non-Stellar network sends a request that silently fails or routes incorrectly.
17. **Form reset on tab switch in `/currency`** – app/currency/page.tsx: Switching from Mint to Burn to International tabs clears all form state. A user who partially fills the mint form, switches to check rates, and switches back loses all input. State should be preserved until explicit reset.
18. **Loading skeleton inconsistent across pages** – app/send/, app/mint/, app/burn/: Some pages show a proper skeleton/spinner during data fetch; others render blank or show stale data briefly. Inconsistent loading UX makes the app feel broken in latency-sensitive environments.
19. **No empty state for zero-transaction history** – app/transactions/page.tsx, app/send/history/page.tsx: When the user has no transactions, the page may show an empty table with no message. Users think the page is broken rather than understanding they have no history yet.
20. **No retry UI on failed API calls** – lib/api/: When `useBalance()`, `ratesApi.getRates()`, or `getMe()` fail, the error is logged to console but no "Retry" button or "Pull to refresh" gesture is offered. Users must reload the entire page.
21. **Toaster position overlaps mobile navigation** – app/layout.tsx: The `<Toaster />` component may render at the bottom of the viewport, overlapping the 5-tab mobile nav bar. Toast action buttons become unclickable when positioned behind the nav.
22. **`useRouter` race condition on fast double-navigation** – Multiple pages: Rapidly tapping two nav tabs in succession can push two routes onto the history stack before the first one completes, corrupting the back-button history on mobile.
23. **i18n route prefix `/[locale]` may cause hydration mismatch** – app/[locale]/: If the locale detection (cookie, accept-language header, geolocation) runs on the server and differs from the client's `navigator.language`, the server-rendered HTML mismatches the client-rendered page, triggering a React hydration error.
24. **Hardcoded strings in pages despite i18n setup** – app/send/, app/mint/, app/business/: Some UI strings ("Send Money", "Confirm", "Insufficient Balance") may be hardcoded in English rather than using translation keys. Non-English locales show a mix of translated and untranslated text.
25. **No offline detection or graceful degradation** – app/: The app makes no use of `navigator.onLine` or `Service Worker` caching. When the user's connection drops mid-flow, API calls fail with opaque network errors and the UI freezes with no "You're offline" indicator.
26. **Form validation runs on every keystroke** – app/send/, app/mint/, app/currency/: Validation errors appear while the user is still typing (e.g., "Invalid amount" when the field is `1.`). Validation should wait until blur or a debounce period to avoid UX frustration.
27. **Mobile keyboard pushes fixed-bottom elements off-screen** – components/mobile-nav.tsx: On iOS Safari, the virtual keyboard pushes the viewport up, but the fixed-bottom mobile nav uses `position: fixed; bottom: 0` and does not account for `visualViewport` API, hiding the nav behind the keyboard.
28. **No password/passcode visibility toggle on auth forms** – app/auth/signin/page.tsx: The passcode input uses `type="password"` with no show/hide toggle. On mobile, users frequently mistype passcodes and can't verify what they entered before submitting.
29. **Mint amount preview overflows on very large numbers** – app/mint/page.tsx, app/currency/page.tsx: The "You'll receive: ACBU X" preview uses a fixed-width container. Large amounts (1,000,000,000+) overflow and break the card layout.
30. **KYC status polling uses fixed interval without backoff** – app/me/kyc/page.tsx: The KYC status page may poll the backend every 3 seconds indefinitely. After approval, it continues polling, wasting bandwidth and server resources. Should stop on terminal status.
31. **RTL (right-to-left) layout untested for Arabic** – app/[locale]/: The app uses `dir` attribute based on locale but no pages have been tested with Arabic or Hebrew text. Tailwind RTL variants (`rtl:`, `ltr:`) are likely missing, causing broken layouts.
88. **`useReducer` state not reset when navigating between different user profiles** – app/admin/users/: Admin pages that use `useReducer` for form state may retain the previous user's form data when navigating from one user edit screen to another, causing accidental data overwrites.
89. **No debounce on search/filter inputs in transaction history** – app/transactions/page.tsx: The transaction filter input fires a state update and re-render on every keystroke. Filtering a list of 1,000+ transactions on every keystroke causes input lag and dropped characters on mid-range devices.
90. **Scroll position lost when toggling between list and grid view** – app/transactions/, app/send/history/: Switching between layout modes resets scroll position. Users lose their place and must scroll back down, causing frustration on long lists.
91. **`React.lazy` not used for heavy page components below the fold** – app/page.tsx: The dashboard imports all chart components, the transaction list, and the quick-actions panel eagerly. Code splitting with `React.lazy` and `Suspense` could reduce the initial bundle by 40%+.
92. **i18n pluralization rules not handling non-English language edge cases** – i18n/: Pluralization for Arabic (6 plural forms) and Slavic languages (3-4 plural forms) use English logic (1=singular, other=plural). Balances and counts display incorrect grammatical forms.
93. **No `inputMode` attribute on numeric form fields** – app/send/page.tsx, app/mint/page.tsx: Amount input fields use `type="text"` without `inputMode="decimal"`. On mobile, the full alphabetic keyboard appears instead of the numeric keypad optimized for decimal entry.
94. **`autocomplete` attributes missing on address and payment forms** – app/send/page.tsx, app/mint/page.tsx: Address, wallet address, and currency fields lack `autocomplete` attributes. Browsers cannot autofill frequently used values, forcing users to re-type or copy-paste each time.
95. **No `aria-live` region for dynamically updating balance display** – app/page.tsx, app/send/page.tsx: When the balance updates via polling or after a transaction, screen readers do not announce the change. Visually impaired users can't perceive balance changes without navigating away and back.
96. **`will-change: transform` used excessively causing GPU memory bloat** – app/globals.css, components/animated-card.tsx: `will-change` is applied to many elements without removing it after the animation completes. Mobile GPUs run out of texture memory, causing choppy animations and, on iOS, a page reload.
97. **`useEffect` dependency array includes objects/arrays created in render** – Multiple hooks: `useEffect(() => {...}, [{ key: value }])` creates a new object every render, causing the effect to run on every render. Infinite re-fetch loops or state update cascades result.
98. **No `ErrorBoundary` around the KYC document upload widget** – app/me/kyc/page.tsx: If the KYC upload widget crashes (camera permission denied, file too large, corrupt image), the entire KYC page shows the Next.js error boundary fallback. The user can't retry without reloading.
99. **`<canvas>` fingerprint preventing bot-detection service from working** – components/security/: If a canvas-based device fingerprinting library is used for fraud detection, it may conflict with the app's canvas usage (charts, animations) or be blocked by Brave/Firefox, causing false-positive fraud flags.
100. **No confirmation dialog when navigating away from multi-step form** – app/send/, app/mint/: If the user has filled a multi-step transfer form and accidentally swipes back (mobile) or clicks a nav link, all input is lost with no "Unsaved changes" warning.
101. **`srcset` and `sizes` attributes missing on responsive images** – app/page.tsx, components/hero.tsx: Responsive images use a single resolution for all viewports. Mobile users download a 1920px-wide hero image scaled to 375px, wasting 500KB+ of data.
102. **No `type="button"` on `<button>` elements inside forms** – app/send/page.tsx, app/mint/page.tsx: Buttons inside forms default to `type="submit"`. A secondary button ("Cancel", "Add Recipient") accidentally submits the form when clicked.
103. **`z-index` values not managed with a design token scale** – app/globals.css, components/: `z-index` values (10, 50, 100, 999, 9999) are scattered across components. A new modal, tooltip, or toast accidentally renders behind the mobile nav or another overlay.
104. **`useSearchParams` accessed without `Suspense` boundary in Next.js** – app/transactions/page.tsx, app/send/page.tsx: Accessing `useSearchParams()` in a component without wrapping in `<Suspense>` triggers a warning in Next.js and can cause the entire page to de-optimize to client-side rendering.
105. **`next/link` `prefetch={false}` not set on logout/deactivation links** – components/user-menu.tsx: The logout link prefetches the logout endpoint page. The prefetch request may trigger a session invalidation or CSRF token consumption, logging the user out before they click.
106. **No `crossorigin="anonymous"` on `next/font` for optimal font loading** – app/layout.tsx: Custom fonts via `next/font/google` may not use `display="swap"` or may block rendering with `display="block"`, keeping text invisible for 3+ seconds on slow connections (FOIT).
107. **`placeholder` text used as label replacement** – app/auth/signin/page.tsx, app/send/: Placeholder text ("Enter amount", "Recipient address") disappears when the user starts typing, leaving no label. Users with cognitive disabilities forget what the field is for mid-entry.
108. **No `prefers-contrast: more` media query support** – app/globals.css: Users with Windows High Contrast Mode or Mac Increased Contrast enabled see the default theme. The app should add high-contrast borders, increase text contrast, and disable transparent backgrounds.
109. **Charts and graphs not keyboard accessible** – components/charts/: Chart components (line, bar, pie) rendered as `<canvas>` or `<svg>` without ARIA roles, keyboard navigation, or textual data alternatives. Screen reader users can't access financial data visualizations.
110. **`pointer-events: none` on loading overlay blocks all interaction** – components/loading-overlay.tsx: A full-screen loading overlay uses `pointer-events: auto` (or none on the overlay but not the content behind). Users on slow connections see the overlay appear and can't even click "Cancel" or navigate away.
111. **No `title` attribute on `<iframe>` elements embedding third-party content** – app/help/, components/support-chat.tsx: Embedded support chat or documentation iframes lack a `title` attribute. Screen reader users navigating by frame hear "frame, frame" with no context.
112. **Text truncation uses `text-overflow: ellipsis` without `title` attribute** – components/transaction-list.tsx, components/account-card.tsx: Long wallet addresses or descriptions are truncated with CSS ellipsis but lack a `title` attribute. There's no way to see the full text without inspecting the DOM.

## Low

32. **Favicon and PWA manifest missing or incomplete** – public/: If `manifest.json` is missing or lacks 192px/512px icons, the "Add to Home Screen" prompt does not appear on Android. The app misses an engagement channel in markets where PWA installation is common.
33. **`aria-label` missing on icon-only buttons** – components/mobile-nav.tsx: Nav bar buttons use only `<Icon />` with no `aria-label`. Screen readers announce "button" with no context, making navigation impossible for visually impaired users.
34. **Color contrast ratio below WCAG AA on disabled buttons** – components/ui/: Disabled primary buttons may use light gray text on a slightly darker gray background, falling below the 4.5:1 contrast ratio required for text.
35. **`tabIndex` not managed in modals and dialogs** – app/: Dialog components may not trap focus within the modal. Tabbing past the last focusable element moves focus to the background page, confusing keyboard-only users.
36. **`<img>` tags missing `alt` attributes** – Multiple pages: Decorative or illustrative images lack `alt=""` and content images lack descriptive `alt` text, failing WCAG 1.1.1 (Non-text Content).
37. **Next.js `Image` component not used for all static images** – app/: Some images use raw `<img>` tags instead of `next/image`. This misses automatic optimization (WebP conversion, lazy loading, responsive sizing) and increases page weight on slow connections.
38. **No skip-to-content link for keyboard navigation** – app/layout.tsx: The layout lacks a visually-hidden "Skip to main content" link as the first focusable element, forcing keyboard users to tab through the entire header and nav on every page.
39. **`prefers-reduced-motion` not respected in animations** – app/globals.css, components/: Tab transitions, spinner animations, and accordion expand/collapse may not check `@media (prefers-reduced-motion: reduce)`, causing discomfort for users with vestibular disorders.
40. **Touch target size below 44x44px on some interactive elements** – Multiple components: Small icon buttons, table row actions, and close buttons may be smaller than the WCAG 2.5.5 minimum of 44 CSS pixels, making them hard to tap accurately on mobile.
41. **`window.matchMedia('(prefers-color-scheme: dark)')` listener not attached** – app/layout.tsx: The dark mode toggle may read the initial preference but not update when the OS-level setting changes while the app is open.
42. **Help page form submits to nowhere** – app/help/page.tsx: The contact form has no backend endpoint — submissions are handled client-side with a fake success toast. Users believe their ticket was filed but support never receives it.
113. **Service Worker not registered for offline caching** – app/layout.tsx, public/sw.js: No Service Worker is registered. The app is fully network-dependent; any connectivity loss renders it unusable instead of gracefully showing cached content.
114. **No `Web Share API` integration for sharing transaction receipts** – app/transactions/, app/send/: Transaction details pages have no "Share" button using `navigator.share()`. Users screenshot and share via other apps, losing structured data (amount, date, hash) that the Share API could provide.
115. **`navigator.vibrate()` pattern for haptic feedback inconsistent across actions** – hooks/use-haptic.ts: Some buttons provide haptic feedback on press; others don't. The inconsistency trains users to expect feedback that doesn't come on critical actions like "Confirm Send."
116. **No `media` attribute on `<link rel="stylesheet">` for print styles** – app/layout.tsx: Print stylesheets are loaded on all devices. On mobile, the print CSS is downloaded and parsed but never used, wasting bandwidth and parse time.
117. **Page title not updated dynamically with route or action context** – app/layout.tsx: The `<title>` tag is static ("ACBU"). Browser history and bookmarks show identical titles for every page, making it impossible to find a specific page in history.
118. **`next/script` `strategy="beforeInteractive"` used for non-critical third-party scripts** – app/layout.tsx: Analytics or chat widgets loaded with `beforeInteractive` block page rendering until they download and execute, adding 500ms+ to Time to Interactive.
119. **No `loading="lazy"` on below-the-fold images** – app/page.tsx, app/me/: All images load eagerly. Below-the-fold images (footer logos, testimonial avatars) compete with above-the-fold content for bandwidth on page load.
120. **`fetchpriority` not specified for critical API calls** – app/page.tsx: The initial balance and user profile fetch requests don't specify `fetchpriority="high"`. Browsers may deprioritize them behind low-priority image and font requests.
121. **`as` prop not used on `next/link` for client-side route transitions** – app/page.tsx, components/nav.tsx: Links to pages that already have data loaded (e.g., navigating to `/send` from `/mint`) trigger a full page transition. The `as` prop can hint Next.js to reuse shared layouts.
122. **No `~` alias for `src/` root, long relative imports used** – Multiple components: Some files use deep relative imports (`../../../../components/ui/button`) instead of the `@/` or `~/` alias. Refactoring directory structure breaks these imports silently.
123. **`console.assert` used in production for invariant checks** – Multiple pages: `console.assert(condition, message)` does not throw or halt execution. A violated invariant is silently logged to the console where no one sees it in production.
124. **`fetch` `.catch()` only logs error without retry or user-facing fallback** – lib/api/client.ts: Failed API calls in the fetch wrapper catch the error and log it but return a rejected promise with no retry logic. Components must implement their own retry, leading to inconsistent behavior across pages.
125. **`useMemo` used on values that don't benefit from memoization** – Multiple hooks: `useMemo(() => value, [dep])` on primitive values or simple calculations adds memoization overhead with no performance benefit. The comparison cost exceeds the recomputation cost.
126. **No `SourceMap` upload to error tracking service** – next.config.mjs: Production builds generate source maps but don't upload them to an error tracking service. Production error stack traces show minified line/column numbers with no way to map to source.
127. **`next-build-id` not configured, cache-busting relies on file hashes alone** – next.config.mjs: Without a custom build ID, deploying a new version while users have the old version open causes chunk load failures. Users see "Loading chunk X failed" and must hard-refresh.
128. **`@tanstack/react-query` or `swr` `staleTime` set to 0 for rarely-changing data** – hooks/use-rates.ts, hooks/use-config.ts: Exchange rates and app configuration that change infrequently (hourly) are re-fetched on every mount. A `staleTime: 5 * 60 * 1000` would eliminate redundant fetches.
129. **No `prefetchDNS` or `preconnect` for third-party domains in `<Head>`** – app/layout.tsx: Third-party domains (analytics, CDN, API) are not pre-connected. The DNS lookup and TCP handshake delay the first request to each domain by 100-300ms.
130. **`next.config.mjs` `compress: true` enabled but CDN also compresses** – next.config.mjs: If the app sits behind a CDN that also gzips, double compression (or compressed-then-uncompressed-then-compressed) can result in corrupted responses or wasted CPU on the origin.
131. **No `crossorigin="use-credentials"` on `<link>` for authenticated font requests** – app/layout.tsx: If fonts are served from a domain requiring cookies, missing `crossorigin="use-credentials"` causes the font request to be sent without cookies and fail.
132. **`aria-expanded` not toggled on collapsible sections** – components/faq.tsx, components/accordion.tsx: Collapsible FAQ sections or accordions use `aria-expanded` but may not update it on toggle. Screen readers always announce the section as "collapsed" even after expanding.
133. **No `role="status"` on live-region balance/count updates** – app/page.tsx: Balance and notification count updates use JS to update the DOM but have no ARIA live region role. Screen reader users don't know the balance changed unless they manually navigate to check.
134. **`<main>` landmark not wrapping primary page content** – app/layout.tsx: The `<main>` element may contain only partial content or be missing entirely. Screen reader landmark navigation (`<main>`, `<nav>`, `<footer>`) is broken or misleading.
135. **No `theme-color` `<meta>` tag for PWA title bar** – app/layout.tsx: The `theme-color` meta tag is missing. On Android Chrome, the address bar/title bar shows the default white/gray instead of matching the app's brand color.
136. **`apple-mobile-web-app-capable` meta tag missing for iOS standalone mode** – app/layout.tsx: The `apple-mobile-web-app-capable` meta tag is absent. iOS users who "Add to Home Screen" get a regular browser tab instead of a standalone app experience.
137. **No `mask-icon` for Safari pinned tabs** – public/: Safari pinned tabs use a monochrome SVG mask icon. Without it, the pinned tab shows the generic letter or a tiny, illegible favicon.

## Trivial

43. **Unused `import` statements in page components** – Multiple pages: Some React components import `useState`, `useEffect`, or icons that are never used, bloating the bundle slightly and triggering ESLint `no-unused-vars` warnings.
44. **Console.log calls in production build** – hooks/use-balance.ts, lib/api/: Debug `console.log` or `console.error` statements persist in production. Browser console shows raw API responses with user data.
